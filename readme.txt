Фіскалізація замовлень з KeyCRM через Multi PRRO Connector: чеки для продажів з Rozetka, Prom, Instagram та власних сайтів в одному місці.

== Опис ==

KeyCRM агрегує замовлення з маркетплейсів (Rozetka, Prom), соцмереж (Instagram, Facebook) та власних сайтів. Цей аддон замикає ланцюжок: він тягне оплачені замовлення з KeyCRM Open API і пробиває фіскальні чеки через ПРРО-сервіси, підключені у плагіні **Multi PRRO Connector** (Вчасно.Каса, Checkbox).

Fiscalization addon for Multi PRRO Connector: pulls paid orders from KeyCRM (an order hub for Rozetka, Prom, Instagram and custom websites) and issues fiscal receipts through the PRRO providers configured in the base plugin.

= Можливості =

* Опитування KeyCRM за розкладом WP-Cron (кожні 5/15/30/60 хв) + кнопка «Запустити зараз»
* Тригер: усі оплачені замовлення (payment_status = paid) або тільки оплачені у вибраних статусах KeyCRM (з довідником статусів прямо на сторінці налаштувань)
* Позиції чека будуються з products замовлення з урахуванням знижок; різниця з grand_total закривається рядком «Доставка» або пропорційним перерахунком — сума чека завжди збігається з сумою замовлення
* Чеки пишуться у спільний журнал «Каса → Чеки» плагіна Multi PRRO Connector (той самий PDF/CSV-експорт)
* ПРРО-сервіс успадковується з налаштувань MPC (основний → резервний), включно з автовідкриттям зміни при чеку
* Опційно: посилання на фіскальний чек дописується у коментар менеджера замовлення KeyCRM (append, без затирання)
* Ідемпотентність: власна таблиця відповідності замовлення ↔ чек з unique key — жодних дублів чеків
* Повага до лімітів KeyCRM: не більше ~55 запитів/хв, обробка 429 з backoff
* API-ключ KeyCRM шифрується у БД тим самим механізмом, що й ключі ПРРО у MPC
* Журнал останніх запусків: скільки перевірено / фіскалізовано / помилок

= Вимоги =

* Активний плагін **Multi PRRO Connector** з налаштованим основним ПРРО-сервісом і відкритою зміною
* API-ключ KeyCRM (кабінет KeyCRM → Налаштування → API)
* PHP 7.4+ (базовий плагін вимагає PHP 8.0)

= Як це працює =

1. Покупець оформлює й оплачує замовлення на Rozetka/Prom/Instagram/сайті
2. Замовлення потрапляє у KeyCRM і отримує payment_status = paid (та/або тригерний статус)
3. Аддон за розкладом тягне оновлені замовлення через GET /order з фільтром updated_between
4. Для кожного нового оплаченого замовлення формується чек і пробивається через ПРРО-адаптери Multi PRRO Connector
5. Чек зʼявляється у «Каса → Чеки», а посилання на нього (опційно) — у коментарі замовлення KeyCRM

== Встановлення ==

1. Встановити й налаштувати Multi PRRO Connector (основний ПРРО-сервіс, «Перевірити звʼязок» = OK)
2. Залити папку аддона у `/wp-content/plugins/mpc-keycrm/` і активувати
3. Меню «Каса → KeyCRM» → ввести API-ключ KeyCRM → Зберегти
4. Натиснути «Завантажити статуси» і вибрати тригер (усі оплачені або конкретні статуси)
5. Натиснути «Запустити зараз» і перевірити журнал запусків та сторінку «Каса → Чеки»

== Часті питання ==

= Чи потрібен WooCommerce? =

Так — його вимагає базовий плагін Multi PRRO Connector. Але самі замовлення беруться з KeyCRM, тож магазин WooCommerce може бути порожнім.

= Що буде, якщо замовлення оновиться повторно? =

Нічого: аддон веде власну таблицю відповідності (unique key по ID замовлення KeyCRM) і додатково перевіряє хеш чека у журналі MPC — чек пробивається рівно один раз.

= Як визначається форма оплати у чеку? =

KeyCRM агрегує оплати з різних джерел без стандартизованого коду форми оплати, тому форма (безготівкова/готівка) задається у налаштуваннях аддона для всіх чеків.

= Чи підтримуються повернення? =

Поки ні — аддон пробиває тільки чеки продажу. Повернення робіть у кабінеті ПРРО-сервісу.
